05 C-Recovery — The Key Unlock

Problem solved: GP-C previously meant "sit out, period." But many C days have improving breadth with real setups available. March 2, 2026: GP-C all day, gap down grinding up, VWAP reclaim and Return Pullback setups present — system said bench.
C-Recovery activates when: Breadth score is between -3 and +3 (GP-C range) AND trajectory is Improving — meaning the last 6 readings show a positive delta ≥ 2.0 or average change ≥ 0.4 per reading.

What's allowed on C-Recovery

Structure/patience setups that don't need institutional follow-through: VWAP-Hold, Pullback-EMA, Failed-BD, BigDogBounce, ReturnPullback, MarketPlay, Rotation-ETF, RS-Override.

What's still blocked on C-Recovery

Momentum setups that need breadth confirmation for follow-through: Momentum, BigDog (full), GapGo, OpeningDrive, DailyBreakout, SecondDay, Scalping.

Sizing constraint

$3-5K base (midpoint $4K). After IVR and beta adjustments, max realistic position is ~$4,500. This enforces Tier 1-3 max automatically — you can't accidentally size up to A-level on a C day.

07 The 5 Code Edits (v90.8 → v90.9)

Edit 1 — SETUP_REGISTRY EXPANDED

Added validGP, minRVOL, nature to all 15 entries. Split BigDog into BigDog + BigDogBounce (16 total). The validGP array is the authoritative source for which setups work in which conditions.

Edit 2 — SUBMODE_SIZING + computePositionSize() BREAKING

Added SUBMODE_SIZING constant, three helpers (_getCurrentSubMode, _checkSetupGP, _getEffectiveSizing), and rewired computePositionSize(). The old hard if (gp === 'C') return blocked is gone — replaced by sizingAdj === 'bench' check which respects C-Recovery.

Edit 3 — evaluateScorecard() Check 1 REWIRED

Check 1 now reads GP_SUB_MODE from ScriptProperties and calls _checkSetupGP(). C-Recovery + structure setup → YELLOW. Caution combos (A-Cooling + Momentum, etc.) → YELLOW.

Edit 4 — getActiveSetups() REWIRED

Reads sub-mode, filters via validGP array instead of hardcoded GP letter checks. C-Recovery tickers surface with gpAllowed: true + reducedSize: true flag. Output now includes caution, nature, minRVOL, subMode.

Edit 5 — getPreTradeData() ENHANCED

Return object now includes gpSubMode, gpSizingAdj, gpSubModeDesc, effective sizing (sub-mode-aware), and full validGP/minRVOL/nature in setupTypes array. Dashboard Phase 3 has everything it needs.

08 How validGP Matching Works

// _checkSetupGP(setupType, subModeLabel, gpLetter)
// 1. If sub-mode label (e.g. 'C-Recovery') is in validGP → allowed
// 2. Else if base letter (e.g. 'A') is in validGP → allowed
//    (but check for caution combos like A-Cooling + Momentum)
// 3. Else → blocked

validGP: ['A', 'B', 'C-Recovery']

A-Surge    → 'A' matches   → allowed
A-Cooling  → 'A' matches   → allowed (may be caution)
B-Steady   → 'B' matches   → allowed
C-Recovery → exact match    → allowed (reduced $3-5K)
C-Flat     → 'C' not in GP  → blocked
D-Persistent → 'D' not in GP → blocked

09 Dependencies & Notes

Requires: Breadth history must be populating in BREADTH_GP_HISTORY ScriptProperty (done by updateMarketTab() since v90.5). classifyTrajectory() needs ≥3 readings to classify — first ~15 minutes of the day will default to "Stable" until enough data accumulates.
Dashboard Phase 3: getPreTradeData() now returns all sub-mode data. When the pre-trade form modal gets built, it can display the sub-mode label, sizing adjustment badge, and the full scorecard with sub-mode-aware Check 1.
SMB Playbook cards (pending): The validGP assignments in this spec are based on Big Dog transcript 1 + the SMB Integration Session notes. As remaining transcripts are processed and playbook cards are built, individual setup validGP arrays may be refined — especially for setups where we only have partial course data (GapGo, OpeningDrive, SecondDay, DailyBreakout).

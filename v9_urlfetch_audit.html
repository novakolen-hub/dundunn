Dynamic Watchlist — How It Works

SOURCES (auto-populated each morning):
  1. In Play Today gappers (pre-market scan, 3%+ gap, RVOL ≥2)
  2. Earnings this week (top revenue not already in Config)
  3. Arete daily email picks (new tickers from morning email)
  4. Manual adds via dashboard (type-to-add input)

LIFECYCLE:
  6:30am  Scanner runs pre-market pass
          → identifies In Play candidates from streaming/gap scan
          → pulls Arete email tickers
          → writes up to 50 tickers to Dynamic tab

  7:30am+ Dynamic tickers join the normal refresh cycle
          → get quotes (free), bars (on partial if conf ≥ threshold)
          → calcConfluence runs, setup detection runs
          → appear in tickerQuotes, Trade Candidates, card labels

  2:00pm  Trading day ends
          → Dynamic tickers with open positions → stay
          → Everything else → cleared for next day

CONFIG TAB STAYS CLEAN:
  Your 375 core tickers don't change.
  Dynamic is a separate parallel universe that adds capacity.

DASHBOARD:
  Trade Candidates merges both sources seamlessly.
  Dynamic tickers show a "⚡" badge so you know they're transient.

What to Trim from Config (424 → 375)

Methodology: Look at the last 30 days of confluence data. Tickers that NEVER reached confluence ≥18 and aren't in any active position or watchlist are candidates for removal. These are the names taking up bar fetches every full scan but never surfacing actionable setups.

Typical trim candidates: low-float names that rarely move with the market, regional ETFs you don't trade, duplicate sector coverage (do you need both XLF AND individual bank stocks?), tickers added speculatively that never triggered.

You don't lose coverage by trimming — any ticker that suddenly matters (earnings gap, news catalyst) enters through the Dynamic watchlist automatically. The Config list is your structural universe. Dynamic is your opportunity scanner.

01 System Philosophy

"The System Protects You. Follow It."

Systematic rules override discretionary decisions. Breadth scoring determines gameplan and position sizing; no trading occurs without system confirmation.

Trading StyleBreadth-confirmed tactical scalping — intraday day trading with supporting swing management
Market Hours7:30am – 2:00pm Arizona (UTC-7, no DST)
Core PrincipleBreadth score drives everything — gameplan, position size, valid setups
ArchitectureDashboard = Research · Chart Wall = Action · Modal = Confirm · TradeStation = Execute

02 Gameplan Framework

The gameplan is determined by the breadth score, which aggregates primary drivers (index internals, A/D line, leadership), regime modifiers (VIX, HYG/TLT, BTC, futures), and sector participation (Growth 8 + Defensive 5).

08 Capital Deployment Guardrails

Position sizing = how big each trade. Guardrails = whether you can take another trade given what's already in play.

09 Trade Management Specs

3.1 Scale-In / Scale-Out TrackingPriority #1

Grouping logic in syncTrades(): match fills to OPEN/PLANNED trades by symbol + account. Scale-in = weighted avg entry update. Scale-out = partial P&L with R-level auto-annotation (e.g., "150@142.50(1.5R)"). Running quantity lifecycle. Pre-trade card shows "ADDING TO EXISTING" when ticker already held.

3.2 Stop Management LifecyclePriority #2

New current_stop field. Four states: INITIAL → BREAKEVEN → TRAILING → (WIDENED = bad). Manual update via modal (v1), broker API polling (v2), auto-inference nag at 1R+ ("move to BE?"). Feeds deployment guardrail BE discount. Enables EOD exit quality grading.

3.4 Unplanned Trade TaggingPriority #4

Toast notification on was_planned=false detection. Lightweight tag form: setup type + stop + reason. EOD prompt for untagged trades. Plan adherence metric in DailyLog. Coaching: "4 planned, 1 unplanned (80% adherence)."

3.5 DT → Swing TransitionPriority #5

When a day trade is working and you want to hold overnight, the system needs to formally transition it — not just let it sit. The trade type changes, the risk profile changes, the stop needs to change, and all of that history needs to be captured.

Transition Triggers

Manual: Click "Convert to Swing" in the dashboard modal for any OPEN DT position.
Auto-detect: runEOD() at 2:00pm AZ finds OPEN trades with trade_type=DT and no exit. Prompts: "Close or convert?"

Conversion Form

Deployment Guardrail Recalculation

On conversion:

• Removed from: Intraday risk budget (dollar risk freed up, DT slot freed up)
• Added to: Swing exposure cap (full market value, INITIAL stop state)
• Validation: if adding to swing exposure would exceed the cap for current GP, show RED warning but allow override — the position already exists, you're just re-categorizing it.

EOD Auto-Detection

If a DT is still open at 2:00pm AZ and user doesn't convert:

⚠️ NVDA (DT) held past close
Entry: $140.00 | Current: $143.50 | P&L: +$1,050
DT Stop: $139.50 | No overnight stop set

[Convert to Swing]   [Plan to Close at Open]

"Plan to Close at Open" keeps trade_type=DT, adds note "Held overnight — plan to close at open." Next morning's dashboard shows: "⚠️ NVDA held overnight. Close or convert."

Analytics Impact

For performance analysis, the system tracks both the label and the behavior:

• trade_type = current label (DT or SW after conversion)
• hold_days = actual calendar days held (0 = true intraday, 1+ = held overnight)
• was_converted = boolean flag for DT→SW transitions
• Analytics can filter by actual behavior: "Show me trades that started as DTs but became swings" — how did those perform vs planned swings?

Nice-to-HavesLower Priority

Daily loss limit: Advisory not hard block. Per-account limits. was_limit_exceeded tracking for coaching patterns.
TOD sizing: 1.0x Open/Core, 0.85x Power, 0.65x Close. Setup exemptions (Scalping, MarketPlay).

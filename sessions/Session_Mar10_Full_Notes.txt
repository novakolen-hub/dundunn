KEY DECISIONS
═══════════════════════════════════════════════════════════════

1. SIZING: % of equity, not fixed dollars
2. BE DISCOUNT: Swings at breakeven count at 50% against cap
3. LOSS LIMITS: Advisory not hard block
4. GP TRANSITIONS: Grandfather existing, only block new entries
5. PAPER TRADES: Exempt from deployment caps
6. PORTFOLIO VIEW: Theme-based clusters as default
7. DT→SW: Formal conversion with full carryover
8. NEWS: 30-min cache, 3 feeds only
9. TIMING: Dynamic DST detection
10. ARETE: Reference only — alignment check, not system override

LESSONS / PROCESS RULES
═══════════════════════════════════════════════════════════════

DASHBOARD PUSH RULE (added to memory):
  Before EVERY push:
  1. Extract JS
  2. Run node -c — if fails, DO NOT PUSH
  3. Verify ALL render functions exist (1 definition each)
  4. Check braces balance
  Never push without ALL checks passing. No exceptions.

FUNCTION BOUNDARY RULE:
  Use brace-counting find_fn_end() for ANY function removal/stubbing.
  NEVER use "\n\nfunction " as boundary marker — it deletes adjacent functions.

ONE PUSH RULE:
  Build locally, verify, show preview, get OK, push ONCE.
  No incremental patches. No "quick fix" pushes.

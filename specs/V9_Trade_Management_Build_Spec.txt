Two new tabs + bridge expansion + dashboard integration

------------------------------------------------------
A. EXECUTIONS TAB (Raw Fills — Auto-Synced)
------------------------------------------------------

One row per broker fill. Fully automated, no manual entry.

Source: TradeStation API + Alpaca API (polled every 1-5 min during market hours)

Fields per row:
  - Exec ID (broker's execution ID, dedup key)
  - Trade # (parent trade ID, assigned by grouping logic)
  - Timestamp (fill time, AZ timezone)
  - Account (TS-DT, TS-SW, ALPACA-PAPER, etc.)
  - Symbol
  - Action (BUY / SELL / SHORT / COVER)
  - Quantity
  - Fill Price
  - Commission
  - Running Qty (calculated: net position after this fill)
  - Order Type (MARKET / LIMIT / STOP)
  - Source (tradestation / alpaca)

Sync logic:
  - Poll TradeStation GET /v3/brokerage/accounts/{id}/orders
    (filter: filled orders since last check)
  - Poll Alpaca GET /v2/orders?status=filled&after={last_check}
  - Dedup by execution ID to prevent double-logging
  - Store last sync timestamp in Script Properties

------------------------------------------------------
B. TRADES TAB (Parent Trades — Grouped & Enriched)
------------------------------------------------------

One row per trade idea. Combines entry/scaling/exit into single record.

Fields per row:

IDENTIFICATION:
  - Trade # (auto-increment)
  - Symbol
  - Account
  - Side (LONG / SHORT)
  - Type (DT / SW / CORE / PAPER)
  - Status (PLANNED / OPEN / CLOSED / CANCELLED)

ENTRY:
  - Entry Time (first fill)
  - Entry Price (avg weighted across scale-ins)
  - Total Shares (max position size reached)
  - Position $ Size
  - Entry Legs (count of buy fills)

EXIT:
  - Exit Time (final fill that closes position)
  - Avg Exit Price (weighted across scale-outs)
  - Exit Legs (count of sell fills)

P&L:
  - Gross P&L ($)
  - Net P&L ($ after commissions)
  - P&L %
  - R-Multiple (P&L / initial risk)
  - Max Favorable Excursion (MFE — best unrealized P&L while open)
  - Max Adverse Excursion (MAE — worst unrealized P&L while open)

PLAN (from pre-trade form OR tagged at EOD):
  - Setup Type (VWAP-Hold, OR-Break, Pullback-EMA, Momentum, Failed-BD, RS-Override, Rotation-ETF)
  - Planned Stop
  - Planned Target
  - Planned Size (from auto-calculator)
  - Dollar Risk (shares × |entry - stop|)
  - Risk % of Account
  - Scale-Out Plan (e.g., "50% at 1R, trail rest")
  - Catalyst/Thesis (one line)

MARKET CONTEXT (auto-snapshotted at entry time):
  - Breadth Score
  - Gameplan (A/B/C/D)
  - Session Phase (MO/MM/PH/MC)
  - VIX
  - VIX Regime
  - Term Structure State (CONTANGO/BACKWARDATION/INVERTED)
  - VIX/VIX3M Ratio
  - VVIX
  - SPY IVR
  - Ticker IVR (from TastyTrade if available)
  - Ticker Beta
  - Expected Move (SPY daily ±$)
  - F&G Score
  - NAAIM Exposure
  - Sector Leader
  - Sector of Symbol

REVIEW (tagged at EOD):
  - Was Planned? (Y/N)
  - Self-Grade (A/B/C/D)
  - Followed System? (Y/N)
  - Notes
  - Lessons

TIMING:
  - Hold Duration (calculated)
  - Days Held (for swings)

C. PRE-TRADE FORM / SCORECARD
------------------------------------------------------

Triggered from dashboard sidebar or scanner menu.
User inputs 4-5 fields, system auto-fills everything else.

USER INPUTS:
  1. Symbol (dropdown from watchlist + type-ahead)
  2. Side (Long / Short)
  3. Setup Type (dropdown: VWAP-Hold, OR-Break, Pullback-EMA,
     Momentum, Failed-BD, RS-Override, Rotation-ETF)
  4. Planned Stop (price)
  5. Planned Target (price) — optional, system suggests

SYSTEM AUTO-CALCULATES & DISPLAYS:

Position Sizing:
  - Current Gameplan → base size ($15-20K / $8-12K / etc.)
  - Ticker IVR adjustment (high IVR = reduce size)
  - Ticker Beta adjustment (high beta = reduce size)
  - Final position size ($)
  - Shares = size ÷ current price
  - Dollar risk = shares × |entry - stop|
  - Risk as % of account

Expected Move Context:
  - SPY expected daily move (±$ and ±%)
  - Ticker expected daily move (SPY EM × beta)
  - Target distance vs expected move ratio
  - Stop distance vs expected move ratio

R-Multiple Targets:
  - 1R price level
  - 2R price level
  - 3R price level
  - Suggested scale plan based on setup type defaults

TRAFFIC LIGHT SCORECARD:
  🟢🟡🔴 Breadth confirms for this side?
  🟢🟡🔴 Setup matches current session phase?
  🟢🟡🔴 IVR supports target distance?
  🟢🟡🔴 Term structure safe? (inverted = red)
  🟢🟡🔴 Not adding correlated exposure? (check existing book)
  🟢🟡🔴 Within daily loss limit?
  🟢🟡🔴 Sector concentration check?
  🟢🟡🔴 Risk per trade within limits?

User sees everything, confirms "ENTER" → creates PLANNED row on Trades tab
When broker fill syncs → matches to planned row, updates to OPEN

If user skips form → broker sync creates UNTAGGED row → tagged at EOD

------------------------------------------------------
D. SCALING LOGIC
------------------------------------------------------

Grouping rules (Executions → Trades):

1. New BUY with no open position in symbol+account = new parent Trade
2. Additional BUY on existing open long = scale-in leg (same parent)
3. SELL against open long = scale-out leg (same parent)
4. When running qty reaches 0 = Trade CLOSED
5. If running qty crosses zero (flip) = close old Trade, open new one
6. Multiple accounts keep separate parents even for same symbol

Trades tab shows summary:
  - "NVDA — Long 100/200 remaining (50% scaled)"
  - "Entry avg: $135.50 | First target hit: $137 (+$150 banked)"
  - "Remaining stop: $134.20 | Current: $138.20 | R on remaining: +2.1R"
  - "Risk-free: ✓" (when banked P&L > remaining risk)

====================================================
ACCOUNT EQUITY & DAILY LOSS LIMIT
====================================================

ACCOUNT EQUITY TRACKING:
  - Pull beginning-of-day equity from TradeStation + Alpaca APIs
  - Store in Script Properties or small "Accounts" config tab
  - Fields: Account ID, Account Label, BOD Equity, Current Equity,
    Daily P&L, Daily P&L %, Last Updated
  - Refreshed on first updateMarketTab trigger each day
  - Required for: position sizing (% of equity), risk % checks,
    daily loss limit enforcement

DAILY LOSS LIMIT:
  - Configurable per account (e.g., DT = -$500, SW = -$1000)
  - Tracked intraday: sum of closed trade P&L + open unrealized P&L
  - Thresholds:
    * Green: daily P&L above -50% of limit
    * Yellow: daily P&L between -50% and -100% of limit
    * Red: daily P&L at or beyond limit
  - When red: pre-trade form shows hard warning, scorecard red
  - Dashboard banner shows running daily P&L at all times
  - EOD: log final daily P&L to a "Daily Results" row for
    long-term tracking (equity curve data)

====================================================
DATA RETENTION & ARCHIVING
====================================================

At 5-10 trades/day with 3-5 executions each:
  - Executions tab: ~15-50 rows/day = ~5,000-12,000 rows/year
  - Trades tab: ~5-10 rows/day = ~1,200-2,500 rows/year

Google Sheets handles up to ~100K rows before performance degrades.
Trades tab is fine for years. Executions tab needs management.

ARCHIVING STRATEGY:
  - Quarterly: move Executions older than 90 days to "Exec Archive" tab
  - Keep Trades tab intact (it's the queryable performance database)
  - Archive function: moveOldExecutions() — manual or scheduled
  - Archived data still accessible for deep analysis, just not
    in the active sync/lookup path
  - Daily Results row (1 per day) lives in its own tab — never archived,
    this becomes the equity curve dataset
